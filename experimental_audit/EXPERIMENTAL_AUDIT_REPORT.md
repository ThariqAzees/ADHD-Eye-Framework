## 1. Executive Verdict
Following a systematic, bottom-up codebase and dataset audit of the ADHD Eye Framework, the following conclusions are established:

1. **Dataset Origin**: The reference dataset (`data/raw/Pupil_dataset.mat` and `data/processed/dataset_features_v1.0.csv`) is **100% synthetic**. The MD5 checksum of the MAT file (`3eaca8c5469793e38b43d2e727c5809b`) does not match the expected Figshare clinical dataset checksum (`d4a1e92c8e125e93831f12797a783d52`). Instead, it was dynamically generated by the local script `dataset/build_dataset.py` using normal distributions.
2. **Authenticity of Clinical Labels**: The ADHD and Control class labels are **wholly simulated** based on hardcoded generative templates. They do not represent real clinical diagnoses.
3. **Reproducibility of ML Performance**: The reported classification accuracies of **Logistic Regression (100%)**, **Random Forest (100%)**, and **XGBoost (98.0%)** are fully reproducible under rigorous nested Stratified $K$-Fold cross-validation.
4. **Cause of Near-Perfect Accuracy**: This performance is the artifact of **zero-overlap distributions** hardcoded in the synthetic generator:
   * **Fixation Instability**: Controls are simulated with a standard deviation of $1.2$ (resulting in a mean feature value of $1.680$ pixels), whereas ADHD is simulated with a standard deviation of $4.8$ (resulting in a mean feature value of $6.700$ pixels). The pooled Cohen's $d$ is **$91.16$**, representing absolute mathematical separation with zero empirical overlap.
   * **Reaction Times**: Controls are simulated around $600\text{ ms}$ (mean $599.258\text{ ms}$), whereas ADHD is simulated around $750\text{ ms}$ (mean $754.499\text{ ms}$). The Cohen's $d$ is **$7.87$** (zero overlap).
5. **Target and Data Leakage**:
   * **Target Leakage**: Yes, the generation mechanism introduces extreme class separability, which behaves like target leakage. Specifically, the features `mean_fixation_stability` and `mean_reaction_time_ms` are deterministic functions of the group labels.
   * **Participant Leakage**: No. The unit of analysis is correctly aggregated at the session level; each row represents a single unique participant ID, so no participant-leakage exists across cross-validation splits.
   * **Preprocessing Leakage**: No. Scaling and imputation are correctly wrapped inside scikit-learn Pipelines, ensuring parameters are fitted only on training folds.
6. **Feature Lineage & Visual Scaling**: The division of fixation instability by $12.0$ (previously claimed to represent "visual degrees calibrated") is mathematically unsupported and arbitrary. The live feature extractor outputs normalized coordinates, whereas the model was trained on these synthetic pixel-deviation values, making them mathematically incompatible.
7. **Domain Shift & Webcam Feasibility**: The pretrained models cannot be transferred directly to webcam gaze-tracking data. The reference database is built on $1000\text{ Hz}$ infrared EyeLink recordings, while the live webcam system operates at $\approx 40\text{ Hz}$ with RGB cameras, introducing massive coordinate offsets, temporal jitter, and lower spatial accuracy.

## 5. Unit of Analysis & Leakage Audits
* **Unit of Analysis**: The dataset contains 50 rows, each matching a unique `subject_id` (`subject_1` to `subject_50`). Therefore, the unit of analysis is the aggregated participant session.
* **Participant Leakage**: 0% leakage risk. Because each row is a separate participant, cross-validation splits partition participants cleanly.
* **Preprocessing Leakage**: 0% leakage risk. Feature scaling (`StandardScaler`) and imputation (`SimpleImputer`) are applied within cross-validation folds, eliminating train-test leakage.
* **Target Leakage**: **100% Leakage Risk**. Although the ML pipeline is technically sound, the features themselves are mathematically linked to the class labels because the data was generated via class-stratified normal distributions without overlap.

## 13. Domain Shift Analysis
There is a fundamental domain shift between the reference dataset and the live webcam tracker:

| Protocol Parameter | Reference Dataset (Rojas-Líbano) | Live System (ADHD Eye) | Domain Shift Severity |
| :--- | :--- | :--- | :--- |
| **Tracking Hardware** | EyeLink 1000 tower-mount physical eye tracker (infrared, fixed chin-rest) | Standard RGB Webcam (uncontrolled environments, user head movement) | **CRITICAL** (pixel offsets vs raw pupil boundaries) |
| **Sampling Rate** | $1000\text{ Hz}$ | Throttled at $\approx 40\text{ Hz}$ | **CRITICAL** (missing micro-saccades and high-frequency pupil changes) |
| **Trial Presentation**| $750\text{ ms}$ dot presentation | $2000\text{ ms}$ dot presentation | **MEDIUM** (impacts cognitive encoding time) |
| **Response Window** | $1500\text{ ms}$ limit | $5000\text{ ms}$ limit | **HIGH** (reaction time features differ systematically) |
| **Trial Count** | 40 trials | 10 trials | **HIGH** (greater variance per subject) |
